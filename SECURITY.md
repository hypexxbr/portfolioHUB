## 🔐 Boas Práticas Adotadas

Este repositório segue as seguintes práticas de segurança:

- **Nenhuma credencial armazenada**: tokens, senhas ou chaves de API nunca são commitados no repositório.
- **Acesso controlado**: apenas o mantenedor principal possui permissão de escrita na branch `main`.
- **Revisão de pull requests**: toda contribuição externa passa por revisão antes do merge.
- **Dependências monitoradas**: o projeto não utiliza dependências externas automatizadas, reduzindo superfície de ataque.

## 🚨 Reportando uma Vulnerabilidade

Se você identificar uma vulnerabilidade de segurança neste projeto, **não abra uma issue pública**.

Entre em contato de forma privada:

📧 **E-mail:** celsofonttes@gmail.com  
📌 **Assunto sugerido:** `[SECURITY] Descrição breve do problema`

### O que incluir no reporte

- Descrição clara da vulnerabilidade
- Passos para reproduzir o problema
- Impacto potencial
- Sugestão de correção (opcional)

### Prazo de resposta

| Etapa | Prazo estimado |
|---|---|
| Confirmação de recebimento | Até 3 dias úteis |
| Avaliação inicial | Até 7 dias úteis |
| Correção e publicação | Até 30 dias (a depender da gravidade) |
